# 研究平台使用验证方案

目标不是只做一个资料下载目录，而是做一个可长期使用的研究工作台：每条资料都保留原始全文，并附中文翻译；搜索结果能回到原始来源、页码或文档位置。

## 资料层

每份资料至少保留四层：

1. 原始文件：PDF、EPUB、HTML、网页快照或图片。
2. 原文全文：按页码或段落切分，不改写原文。
3. 中文翻译：与原文切分单元一一对应，标记翻译状态。
4. 研究元数据：来源、标题、日期、人物、组织、地点、事件、可靠性等级、备注。

中文翻译不能覆盖原文。翻译只是一层阅读辅助，所有研究结论必须能回到原文和出处。

## 搜索方式

平台应支持三类搜索：

1. 原文搜索：如 `Democratic League`、`Federation of Chinese Democratic Parties`、`Lo Lung-chi`。
2. 中文搜索：如 `民盟`、`昆明暗杀`、`政治协商会议`、`罗隆基`。
3. 混合检索：中文查询也能返回英文原文；英文查询也显示中文翻译。

搜索结果应显示：

- 标题、日期、来源。
- 命中的原文片段。
- 对应中文翻译。
- 页码或文档级引用。
- 原始链接和本地快照路径。
- 翻译状态：机器初译、人工校订、待复核。

## 阅读方式

单篇文档阅读页建议采用左右并排：

- 左侧：原文全文。
- 右侧：中文翻译。
- 顶部：来源、日期、卷册、文档号、原始链接。
- 旁边：人物、组织、地点、主题标签。

对扫描 PDF，应按 PDF 页码显示；对 FRUS 这类 EPUB/HTML，如果官方给了页码锚点，优先显示官方页码；没有页码锚点时显示文档级引用。

## 翻译状态

建议使用以下状态：

- `machine-draft`: 机器初译。
- `human-reviewed`: 人工校订。
- `sample-review-needed`: 样本翻译，待复核。
- `needs-translation`: 尚未翻译。

后续如果使用模型批量翻译，应保存模型、提示词、生成时间，避免以后无法判断译文来源。

## 当前验证

已用 FRUS `frus1946v09/d735` 做了样本：原文按片段保留，中文翻译写入 `translations` 表，双语搜索脚本可以同时返回原文和中文翻译。

示例：

```bash
python3 scripts/add_translation_sample.py
python3 scripts/search_bilingual.py '昆明暗杀' --limit 5
python3 scripts/search_bilingual.py 'Democratic League' --limit 5
```

这验证了后续平台的基本使用方式：先搜，再看双语片段，最后点回原始来源和页码。
